Export du schema de prop avec toutes ses données :

- Creer le repertoire sur l'OS et donner les accès pour qu'oracle puisse lire et écrire dedans.
- Indiquer le repertoire via SQLPLUS : CREATE OR REPLACE DIRECTORY prop_dump_dir AS '/home/aurelien/Documents/M1IFI/dump_oracle/';
- Donner les accès à l'utilisateur qui veux réaliser l'import/export : grant read, write on DIRECTORY prop_dump_dir to prop
- Lancer depuis un terminal ou console : expdp prop/passprop schemas=prop directory=prop_dump_dir dumpfile=prop.dmp logfile=prop.log

Import du schema de prop avec toutes ses données (attention n'est importable que si prop est le propriétaire c'est à dire que celui qui réalise l'import importe son propre schéma qu'il avait exporté précédemment)

- Lancer depuis un terminal ou console : impdp prop/passprop directory=prop_dump_dir dumpfile=prop.dmp logfile=prop.log TABLE_EXISTS_ACTION=replace
(on specifie l'action pour TABLE_EXTSTS_ACTION a replace au cas ou les tables existent déjà en base.
